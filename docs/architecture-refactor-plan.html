01The target architecture

A four-layer onion. Time and the database are the two substrate seams everything injects. Above them sit pure, infra-free domain kernels (the heart of the new test suite), then thin data accessors over the one database port, then the external-I/O edges, then presentation. The four giant files dissolve onto these layers last.

02The 24 modules

Every module collapses a confirmed N-caller duplication or is a testability keystone — nothing speculative. The deletion test governs the count: delete the module and the same logic must reappear across its callers. Each card opens to its interface, the current code it absorbs, why it's deep, and how it gets tested.

Layer 0 · Substrate seams

PerthClock

Layer 0 · time · keystone
in-processunanimousS

The single injectable source of "now in Perth" (AWST, UTC+8, no DST) and the calendar fields the whole app reasons about: Perth-local date, day-of-week, minutes-of-day, and the YYYY-MM-DD "today" string. Every time-dependent module takes now as a parameter and becomes deterministic. This is the keystone — nothing time-dependent is testable today because each module reads new Date() internally via 3 different idioms.

Interface · absorbs · why deep · tests
InterfaceperthNow(now = new Date()): { date, dayOfWeek 0..6, minutesOfDay, ymd }. Critical invariant (red-team fix): derive fields from absolute UTC — getUTCHours()+8 with day-wraparound (or Intl parts with timeZone:'Australia/Perth') — never new Date(now.getTime()+8h) then local getters, which double-shifts in a non-UTC browser (Perth users run UTC+8, not UTC). now is the only time input.
Absorbs
  • getPerthNow in happyHourLive.ts:43 (the safe pattern to generalise)
  • the different Intl idiom in happyHour.ts:108–114
  • the now+8h block in pint-of-the-day/route.ts:25
  • inline Perth-time idioms in TonightsMoves, DiscoverClient, PintOfTheDay, race-meets
Why deepA ~15-line pure function behind which the entire app's time reasoning lives, replacing 5+ divergent timezone idioms (two of which disagree on host-zone safety). Delete it and the same getUTC/Intl wraparound reappears in happy-hour, freshness, pint-of-the-day, sun overlay, and the Andrew scheduler.
TestsRun the same suite under TZ=Australia/Perth AND TZ=America/New_York, asserting identical output — the regression that catches the double-shift bug. No I/O.

Absorbs

03Ports & seams — exactly three

A port is justified only where two real adapters exist AND there's non-trivial logic across the seam. That's true in exactly three places. Everywhere else (ElevenLabs, web-push, weather, geolocation) the rule is: extract the pure core, call fetch/webpush/navigator directly, and monkeypatch in a single edge test. This honours all four prior architectural rejections.

SupabaseGateway — the one database port

ProductioncreateClient(url, key) against the live Supabase project (PostgREST)
Testa typed fake SupabaseClient implementing only the builder methods the accessors use, returning canned {data,error}

The single seam where two real adapters exist and there's non-trivial logic to test across it (every Layer-2 accessor). 16 inline createClient copies + a leaked anon JWT in 14 files collapse into it. The second adapter must be concrete before Layer 2 is built — PGLite is explicitly rejected (raw Postgres ≠ PostgREST wire protocol).

PerthClock — the time seam

ProductionperthNow() with now = new Date() at the edge
TestperthNow(fixedDate) — a frozen Date injected by every downstream unit test

The keystone that converts the entire time-dependent surface (happy-hour, freshness, pint-of-the-day, sun overlay, Andrew scheduler) from untestable to deterministic. The variation across the seam is exercised by every Layer-1 test.

GeoRouting RNG — the shuffle seam

ProductionMath.random for the Fisher-Yates crawl shuffle + lucky-dip course pick
Testa seeded/deterministic RNG injected into planRoute and the lucky-dip picker

Without it, CrawlPlanner.planRoute(shuffle:true) and GolfScoring's lucky-dip are untestable. Narrow and justified — the only place randomness varies.

05The principles that hold it together

These are the rules every PR in the migration is checked against — the reason the module count is what it is, and the guardrails that keep a live site safe.

  1. The deletion test governs the count, not a target number. Every module must collapse a confirmed N-caller duplication or be a testability keystone. Three first-draft modules failed this and were cut/shrunk: DesignTokens (its verdict→class maps each have ≤1 live caller), ChartPrimitives (→ a ~6-line helper + in-place token fix), and SiteChrome's nav-union (deliberately different IA, not copy-paste).
  2. Exactly one true-external port: SupabaseGateway. A port is justified only where two real adapters exist AND there's non-trivial logic across the seam. Don't inject elevenlabs/places/web-push/weather/geolocation ports — extract the pure core, call fetch/webpush/navigator directly, monkeypatch in a single edge test. Honours all four prior rejections.
  3. Inject the clock everywhere; derive Perth time from absolute UTC. Nothing reads new Date() internally — every time-dependent module takes now = perthNow().date, computed from getUTC*+8h with wraparound, never new Date(+8h) then local getters (which double-shifts in a non-UTC browser). Tested under two host timezones.
  4. Behaviour-preserving on a live SEO site: pin before you move. The site auto-deploys from main. Before relocating any identity function or collapsing any duplicated output, land a characterization/snapshot test that pins current output (slug strings, scheduleLabel, mapper output, suburb-page <title>/JSON-LD). Any diff is a release blocker.
  5. Structured columns are the source of truth; free text is derived. The happy-hour engine consumes structured days/start/end; the .happyHour string is produced by scheduleLabel(), not parsed as primary input. parseScheduleText is a fallback only when the structured columns are null.
  6. Separate pure from impure so the pure part is unit-testable. Verdict/rule kernels (PriceRules, PubStats, GeoRouting, the HMAC parse, validation bands, weather/race mapping, route-planning, golf-scoring, week-over-week delta) are pure and infra-free; the thin I/O edge calls fetch/DB directly. This is what lets a ~0-test repo gain a wide unit surface — the whole point of the exercise.
  7. Surgical, config-aware changes. Two changes are config-coupled, not pure refactors, and must verify live config first: removing the ElevenLabs HMAC bypass (confirm the dashboard sends HMAC) and the serviceClient() throw-on-missing (confirm the key in every write runtime). Account for vercel.json — it holds a third redirect layer + two live crons; keep cron route files thin since Vercel invokes them by URL.
  8. Core-extraction first, shell-split last. For the four giants, extract the deep pure core (leverage win, unblocks tests) before splitting the dumb shell by phase/tab (locality win only). Don't invent reusable component libraries beyond those that earn their keep. Geolocation stays a direct hook.

How this was built

Produced by a 17-agent multi-agent workflow over the live codebase, built on a prior architecture review whose confirmed wins and explicit rejections seeded every agent.
7 surveyors → 4 independent architects → 1 reconciler → 4 red-teamers → 1 finalizer

The four architects each designed a full decomposition under a different governing principle — domain-driven, ports-and-adapters, minimal-churn, and testability-first — and the reconciler kept only what they agreed on, resolving the contested boundaries with engineering rationale. The red-team raised 22 blockers across depth, over-abstraction, missing-coupling, and migration-safety; all 22 were resolved into the plan above (it shrank DesignTokens and ChartPrimitives, promoted CrawlPlanner/GolfScoring/WatchlistStore/AdminDashboardData from prose to owned modules, and fixed the timezone double-shift, the HMAC-bypass sequencing, and the suburb N+1). Zero were genuine business blockers — the two on the first screen are product/ops calls only you can make.

Consensus levels: unanimous = all four architects proposed it independently · majority = two or three · resolved = contested, settled by the red-team. Every file:line reference was verified against source during the run.
